Class: Queue

ROSQUEUE. Queue

new Queue(options)

Communicates with rms_queue_manager node to handle users. Developed to allow multiple users to visit an RMS interface to control a single robot Users will be placed into a queue giving the first user control of the robot for a certain amount of time When that user leaves, or their time ends, they will be kicked out and the next user gains control
Parameters:
Name Type Description
options object with the following keys * ros - the ros ROSLIB.Ros connection handle * userId - the ID of the user, used to distinguish users * studyTime - time in minutes that the study is conducted for
